# Skill: OpenSpec — Explorar Ideia

Modo thinking-partner para clarificar uma ideia **antes** de virar um change no OpenSpec. Não crie nenhum arquivo.

## Instruções

1. Pergunte o necessário para fixar o comportamento (happy + sad paths, edge cases, ownership/auth, error codes).
2. Identifique qual capability/spec a ideia toca (nova vs existente) — verifique `openspec/specs/` se presente.
3. Esboce os cenários GIVEN/WHEN/THEN informalmente para ver se algum ainda está vago.
4. Termine com uma recomendação: está pronto para propor um change, ou precisa de mais reflexão?

Mantenha conversacional. O objetivo é um escopo nítido e testável — sem artefatos ainda.
